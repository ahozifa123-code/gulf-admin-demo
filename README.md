# 🏛 نظام الخليج للخدمات الإدارية
## نظام محاسبة سطح المكتب - شبكة LAN الداخلية

---

## 📋 متطلبات التشغيل

- **Node.js** (الإصدار 16 أو أحدث) — تحميل من: https://nodejs.org
- متصفح حديث (Chrome, Edge, Firefox)
- شبكة LAN داخلية (للوصول من أجهزة متعددة)

---

## 🚀 طريقة التشغيل

### على Windows:
1. افتح مجلد النظام
2. انقر نقراً مزدوجاً على ملف: **تشغيل-السيرفر.bat**
3. سيفتح المتصفح تلقائياً

### على Linux/Mac:
```bash
chmod +x start.sh
./start.sh
```

### يدوياً (أي نظام):
```bash
npm install
node server.js
```

---

## 🌐 الوصول من أجهزة الشبكة

بعد تشغيل السيرفر، يظهر في نافذة الأوامر:
```
✅ السيرفر يعمل على المنفذ: 3000
🖥  هذا الجهاز:      http://localhost:3000
🌐 الشبكة الداخلية:  http://192.168.1.X:3000
```

**على أي جهاز آخر في الشبكة:**
افتح المتصفح واكتب: `http://192.168.1.X:3000`
(استبدل X بالعنوان الظاهر في النافذة)

## 📁 هيكل الملفات

```
gulf-admin-lan/
├── server.js              ← السيرفر الرئيسي
├── package.json           ← إعدادات المشروع
├── تشغيل-السيرفر.bat     ← تشغيل سريع (Windows)
├── start.sh               ← تشغيل سريع (Linux/Mac)
├── data/
│   └── gulf_admin.db      ← قاعدة البيانات (تُنشأ تلقائياً)
└── public/
    └── index.html         ← الواجهة الأمامية
```

---

## 💾 قاعدة البيانات والنسخ الاحتياطية

- البيانات تُحفظ في ملف `data/gulf_admin.db`
- **النسخة الاحتياطية:** اذهب للإعدادات ← "نسخة احتياطية" لتنزيل JSON
- احتفظ بنسخة من مجلد `data/` بانتظام

---

## 🔧 تغيير المنفذ

إذا كان المنفذ 3000 مستخدماً:
```bash
PORT=8080 node server.js
```
أو عدّل الرقم في `server.js` السطر: `const PORT = 3000;`

## 🆘 استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| "لا يمكن الاتصال بالسيرفر" | تأكد من تشغيل server.js أولاً |
| لا يظهر في الأجهزة الأخرى | تحقق من Firewall → اسمح بالمنفذ 3000 |
| "Port already in use" | غيّر المنفذ أو أوقف البرنامج الآخر |
| البيانات لا تُحفظ | تأكد من صلاحيات الكتابة في مجلد data/ |
